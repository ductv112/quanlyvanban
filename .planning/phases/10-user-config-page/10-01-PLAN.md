### 2. POST `/api/ky-so/tai-khoan`
Body:
```json
{ "user_id": "0123456789", "credential_id": "CRED_001" }
```
- Validation: `user_id` non-empty string ≤ 200 chars. `credential_id` optional (bắt buộc cho MYSIGN_VIETTEL, null cho SMARTCA_VNPT).
- Business rules:
  - Lấy active provider qua `getActiveProviderWithCredentials()` — nếu null → 400 "Admin chưa kích hoạt provider ký số nào..."
  - Nếu `active.provider.code === 'MYSIGN_VIETTEL'` và `credential_id` rỗng → 400 "Vui lòng chọn chứng thư số"
  - Upsert qua `staffSigningConfigRepository.upsert` với `is_verified=false`, `last_verified_at=null`, `certificate_*=null` (reset verify status mỗi lần save; user phải bấm "Kiểm tra" để verify lại).
  - Trường certificate_data/subject/serial pass `null` → SP COALESCE giữ nguyên cert cũ nếu có (xem SP `fn_staff_signing_config_upsert` line 391-393). **Nhưng cần reset is_verified=false** để UI hiển thị "Chưa xác thực" cho đến khi user verify lại.

Response 201: `{ success: true, message: "Lưu cấu hình thành công" }`
Response 400: validation errors, provider not active, missing credential_id for MySign

### 3. POST `/api/ky-so/tai-khoan/certificates`
Body:
```json
{ "user_id": "CMT_123456" }
```
- Chỉ MYSIGN_VIETTEL cần endpoint này (SmartCA listCertificates vẫn work nhưng usually trả 1 cert; MySign có thể nhiều).
- Lấy active provider — nếu null → 400
- Call `active.provider.listCertificates(active.credentials, { userId: body.user_id })`.
- Trả về danh sách cert đã normalize (shape `CertificateInfo[]`).

Response 200:
```json
{
  "success": true,
  "data": {
    "certificates": [
      {
        "credential_id": "CRED_001",
        "subject": "CN=Nguyễn Văn A, O=Sở Nội vụ",
        "serial_number": "0A1B2C",
        "valid_from": "2025-01-01T00:00:00.000Z",
        "valid_to": "2027-01-01T00:00:00.000Z",
        "status": "active"
      }
    ]
  }
}
```
Response 400: provider not active, empty user_id
Response 502: provider error (network, auth)

Transform: map `credentialId` → `credential_id`, `serialNumber` → `serial_number`, `validFrom` → `valid_from`, `validTo` → `valid_to` (snake_case ra FE). KHÔNG trả `certificateBase64` (không cần ở list UI, tiết kiệm payload).

### 4. POST `/api/ky-so/tai-khoan/verify`
Body: không có body (hoặc empty `{}`)
- Đọc config hiện tại qua `staffSigningConfigRepository.get(staffId, active.provider.code)`.
- Nếu chưa có config → 400 "Vui lòng lưu cấu hình trước khi kiểm tra"
- Call `active.provider.listCertificates(active.credentials, { userId: config.user_id, credentialId: config.credential_id ?? null })`.
  - SmartCA: API trả 1 cert → dùng luôn
  - MySign: API trả list → filter theo `config.credential_id` (match `credentialId` field), nếu không tìm thấy → verify FAIL
- Nếu có cert khớp:
  - Upsert lại config với `is_verified=true`, `last_verified_at=new Date().toISOString()`, `certificate_subject=cert.subject`, `certificate_serial=cert.serialNumber`, `certificate_data=cert.certificateBase64`, `last_error=null`.
  - Return 200 với `{ verified: true, certificate_subject, cert_valid_to, last_verified_at }`
- Nếu listCertificates throw hoặc không có cert khớp:
  - Upsert với `is_verified=false`, `last_verified_at=null` (giữ user_id/credential_id), `last_error=msg`.
  - Return 200 với `{ verified: false, message }` (KHÔNG throw 500 — verify fail là expected state, không phải server error).
- Nếu network/provider error thực sự (unreachable) → 502.

Response 200 verified:
```json
{
  "success": true,
  "data": {
    "verified": true,
    "certificate_subject": "CN=Nguyễn Văn A",
    "cert_valid_to": "2027-01-01T00:00:00.000Z",
    "last_verified_at": "2026-04-21T10:20:00.000Z"
  }
}
```
Response 200 not verified (expected — cert not found or expired):
```json
{ "success": true, "data": { "verified": false, "message": "Không tìm thấy chứng thư khớp credential_id đã chọn" } }
```
Response 502: network/provider error

```typescript
/**
 * Route: /api/ky-so/tai-khoan* — User-level signing config (CFG-05, CFG-06).
 *
 * 4 endpoints, authenticate only (NO requireRoles — mọi user quản lý config cá nhân):
 *   GET    /             — lấy config hiện tại + active provider metadata
 *   POST   /             — upsert config user (user_id + optional credential_id)
 *   POST   /certificates — list cert từ provider (MySign — fetch CTS)
 *   POST   /verify       — verify config bằng listCertificates + lưu cert snapshot
 *
 * SECURITY: staffId LUÔN lấy từ JWT (req.user.staffId), KHÔNG BAO GIỜ từ body — ngăn
 *   user A cập nhật config của user B (T-10-01 Tampering).
 *
 * Mount trong server.ts — BEFORE /api/ky-so/cau-hinh (giữ pattern longer-prefix-wins
 * đã có; vì requireRoles guard cho /cau-hinh áp dụng ngay trên mount đó, tai-khoan
 * authenticate-only mount riêng không leak admin guard).
 */

import { Router } from 'express';
import type { Request, Response } from 'express';
import type { AuthRequest } from '../middleware/auth.js';
import { staffSigningConfigRepository } from '../repositories/staff-signing-config.repository.js';
import { getActiveProviderWithCredentials } from '../services/signing/providers/provider-factory.js';
import type { CertificateInfo } from '../services/signing/providers/provider.interface.js';
import { handleDbError } from '../lib/error-handler.js';

const router = Router();

// ============================================================================
// Helpers
// ============================================================================

function isNonEmptyString(v: unknown, maxLen = 200): v is string {
  return typeof v === 'string' && v.trim().length > 0 && v.trim().length <= maxLen;
}

/** Map CertificateInfo (camelCase) → snake_case cho FE */
function toClientCert(c: CertificateInfo) {
  return {
    credential_id: c.credentialId,
    subject: c.subject,
    serial_number: c.serialNumber,
    valid_from: c.validFrom,
    valid_to: c.validTo,
    status: c.status,
  };
}

// ============================================================================
// 1. GET / — lấy active provider metadata + user config
// ============================================================================

router.get('/', async (req: Request, res: Response) => {
  try {
    const { staffId } = (req as AuthRequest).user;

    const active = await getActiveProviderWithCredentials();
    if (!active) {
      res.json({
        success: true,
        data: {
          active: null,
          config: null,
          message:
            'Admin chưa kích hoạt provider ký số nào. Vui lòng liên hệ Quản trị viên.',
        },
      });
      return;
    }

    const providerCode = active.provider.code;
    const providerName =
      providerCode === 'SMARTCA_VNPT' ? 'SmartCA VNPT' : 'MySign Viettel';

    const config = await staffSigningConfigRepository.get(staffId, providerCode);

    res.json({
      success: true,
      data: {
        active: {
          provider_code: providerCode,
          provider_name: providerName,
          base_url: active.credentials.baseUrl,
        },
        config: config
          ? {
              staff_id: config.staff_id,
              provider_code: config.provider_code,
              user_id: config.user_id,
              credential_id: config.credential_id,
              certificate_subject: config.certificate_subject,
              certificate_serial: config.certificate_serial,
              is_verified: config.is_verified,
              last_verified_at: config.last_verified_at,
            }
          : null,
      },
    });
  } catch (error) {
    handleDbError(error, res);
  }
});

// ============================================================================
// 2. POST / — upsert user config
// ============================================================================

router.post('/', async (req: Request, res: Response) => {
  try {
    const { staffId } = (req as AuthRequest).user;
    const body = req.body ?? {};

    if (!isNonEmptyString(body.user_id, 200)) {
      res.status(400).json({ success: false, message: 'Vui lòng nhập user_id' });
      return;
    }

    const active = await getActiveProviderWithCredentials();
    if (!active) {
      res.status(400).json({
        success: false,
        message:
          'Admin chưa kích hoạt provider ký số nào. Vui lòng liên hệ Quản trị viên.',
      });
      return;
    }

    const providerCode = active.provider.code;

    // MySign bắt buộc có credential_id (multi-cert); SmartCA thì null OK
    let credentialId: string | null = null;
    if (providerCode === 'MYSIGN_VIETTEL') {
      if (!isNonEmptyString(body.credential_id, 200)) {
        res.status(400).json({
          success: false,
          message:
            'Vui lòng chọn chứng thư số (bấm "Tải danh sách CTS" để lấy danh sách)',
        });
        return;
      }
      credentialId = body.credential_id.trim();
    } else if (isNonEmptyString(body.credential_id, 200)) {
      // SmartCA có thể có credential_id nhưng không bắt buộc
      credentialId = body.credential_id.trim();
    }

    const result = await staffSigningConfigRepository.upsert({
      staffId,
      providerCode,
      userId: body.user_id.trim(),
      credentialId,
      // Pass null cho cert fields — SP COALESCE giữ nguyên cert cũ nếu có
      certificateData: null,
      certificateSubject: null,
      certificateSerial: null,
      // Reset verify status — user phải bấm "Kiểm tra" để verify lại sau mỗi lần save
      isVerified: false,
      lastVerifiedAt: null,
      lastError: null,
    });

    if (!result.success) {
      res.status(400).json({ success: false, message: result.message });
      return;
    }

    res.status(201).json({
      success: true,
      message: 'Lưu cấu hình thành công. Vui lòng bấm "Kiểm tra" để xác thực.',
    });
  } catch (error) {
    handleDbError(error, res);
  }
});

// ============================================================================
// 3. POST /certificates — fetch cert list từ provider (MySign — Tải danh sách CTS)
// ============================================================================

router.post('/certificates', async (req: Request, res: Response) => {
  try {
    const body = req.body ?? {};

    if (!isNonEmptyString(body.user_id, 200)) {
      res.status(400).json({ success: false, message: 'Vui lòng nhập user_id' });
      return;
    }

    const active = await getActiveProviderWithCredentials();
    if (!active) {
      res.status(400).json({
        success: false,
        message:
          'Admin chưa kích hoạt provider ký số nào. Vui lòng liên hệ Quản trị viên.',
      });
      return;
    }

    let certs: CertificateInfo[];
    try {
      certs = await active.provider.listCertificates(active.credentials, {
        userId: body.user_id.trim(),
      });
    } catch (err: unknown) {
      const msg = err instanceof Error ? err.message : 'Lỗi không xác định';
      res.status(502).json({
        success: false,
        message: 'Không lấy được danh sách chứng thư: ' + msg,
      });
      return;
    }

    res.json({
      success: true,
      data: {
        certificates: certs.map(toClientCert),
      },
    });
  } catch (error) {
    handleDbError(error, res);
  }
});

// ============================================================================
// 4. POST /verify — verify config + lưu cert snapshot
// ============================================================================

router.post('/verify', async (req: Request, res: Response) => {
  try {
    const { staffId } = (req as AuthRequest).user;

    const active = await getActiveProviderWithCredentials();
    if (!active) {
      res.status(400).json({
        success: false,
        message:
          'Admin chưa kích hoạt provider ký số nào. Vui lòng liên hệ Quản trị viên.',
      });
      return;
    }

    const providerCode = active.provider.code;
    const config = await staffSigningConfigRepository.get(staffId, providerCode);
    if (!config) {
      res.status(400).json({
        success: false,
        message: 'Vui lòng lưu cấu hình trước khi kiểm tra',
      });
      return;
    }

    // Gọi listCertificates; nếu throw (network) → 502, nếu throw (auth fail) → vẫn 502
    let certs: CertificateInfo[];
    try {
      certs = await active.provider.listCertificates(active.credentials, {
        userId: config.user_id,
        credentialId: config.credential_id,
      });
    } catch (err: unknown) {
      const msg = err instanceof Error ? err.message : 'Lỗi không xác định';
      // Persist failure (non-fatal nếu repo cũng lỗi)
      try {
        await staffSigningConfigRepository.upsert({
          staffId,
          providerCode,
          userId: config.user_id,
          credentialId: config.credential_id,
          certificateData: null,
          certificateSubject: null,
          certificateSerial: null,
          isVerified: false,
          lastVerifiedAt: null,
          lastError: msg,
        });
      } catch {
        /* ignore */
      }
      res.status(502).json({
        success: false,
        message: 'Không kết nối được provider: ' + msg,
      });
      return;
    }

    // Tìm cert khớp:
    //   SmartCA: list thường chỉ 1 cert → dùng cert đầu tiên
    //   MySign: match theo credentialId
    let matched: CertificateInfo | undefined;
    if (providerCode === 'MYSIGN_VIETTEL') {
      matched = certs.find((c) => c.credentialId === config.credential_id);
    } else {
      matched = certs[0];
    }

    if (!matched) {
      const errMsg =
        providerCode === 'MYSIGN_VIETTEL'
          ? 'Không tìm thấy chứng thư khớp credential_id đã chọn. Vui lòng tải lại danh sách CTS.'
          : 'Không tìm thấy chứng thư nào cho user_id này';

      await staffSigningConfigRepository.upsert({
        staffId,
        providerCode,
        userId: config.user_id,
        credentialId: config.credential_id,
        certificateData: null,
        certificateSubject: null,
        certificateSerial: null,
        isVerified: false,
        lastVerifiedAt: null,
        lastError: errMsg,
      });

      res.json({
        success: true,
        data: { verified: false, message: errMsg },
      });
      return;
    }

    const nowIso = new Date().toISOString();
    const upsertResult = await staffSigningConfigRepository.upsert({
      staffId,
      providerCode,
      userId: config.user_id,
      credentialId: config.credential_id,
      certificateData: matched.certificateBase64,
      certificateSubject: matched.subject,
      certificateSerial: matched.serialNumber,
      isVerified: true,
      lastVerifiedAt: nowIso,
      lastError: null,
    });

    if (!upsertResult.success) {
      res.status(400).json({ success: false, message: upsertResult.message });
      return;
    }

    res.json({
      success: true,
      data: {
        verified: true,
        certificate_subject: matched.subject,
        cert_valid_to: matched.validTo,
        last_verified_at: nowIso,
      },
    });
  } catch (error) {
    handleDbError(error, res);
  }
});

export default router;
```
