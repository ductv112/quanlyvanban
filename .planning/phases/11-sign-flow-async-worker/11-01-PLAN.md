**Part 2: fn_attachment_finalize_sign** — worker call sau khi embed signature + upload MinIO.
Signature: `(p_attachment_id BIGINT, p_attachment_type VARCHAR, p_signed_file_path VARCHAR, p_sign_provider_code VARCHAR, p_sign_transaction_id BIGINT)` returns `TABLE(success BOOLEAN, message TEXT)`.

Body — IF/ELSIF for 4 types:
```sql
IF p_attachment_type = 'outgoing' THEN
  UPDATE edoc.attachment_outgoing_docs
    SET is_ca = TRUE, ca_date = NOW(),
        signed_file_path = p_signed_file_path,
        sign_provider_code = p_sign_provider_code,
        sign_transaction_id = p_sign_transaction_id
    WHERE id = p_attachment_id;
ELSIF p_attachment_type = 'drafting' THEN
  UPDATE edoc.attachment_drafting_docs SET ... WHERE id = p_attachment_id;
ELSIF p_attachment_type = 'handling' THEN
  UPDATE edoc.attachment_handling_docs SET ... WHERE id = p_attachment_id;
ELSIF p_attachment_type = 'incoming' THEN
  UPDATE edoc.attachment_incoming_docs SET ... WHERE id = p_attachment_id;
ELSE
  RETURN QUERY SELECT FALSE, 'Loại file đính kèm không hợp lệ'::TEXT; RETURN;
END IF;

IF NOT FOUND THEN
  RETURN QUERY SELECT FALSE, 'Không tìm thấy file đính kèm'::TEXT; RETURN;
END IF;

RETURN QUERY SELECT TRUE, 'Ký số thành công'::TEXT;
```

**Part 3: fn_attachment_can_sign** — permission check before enqueue.
Signature: `(p_attachment_id BIGINT, p_attachment_type VARCHAR, p_staff_id INT)` returns `TABLE(can_sign BOOLEAN, reason TEXT, file_path VARCHAR, file_name VARCHAR)`.

Logic:
- Lookup attachment by (id, type) → if not found, can_sign=FALSE reason='Không tìm thấy file đính kèm'
- If `is_ca = TRUE`, can_sign=FALSE reason='File đã được ký số' (for types that have is_ca column)
- For attachment_type = 'incoming': can_sign=FALSE reason='Không được ký số văn bản đến' (VB đến là input, không ký)
- For 'outgoing' / 'drafting': JOIN parent doc, get signer VARCHAR + created_by INT + approver VARCHAR. Allow if:
  - `p_staff_id = created_by`, OR
  - `LOWER(UNACCENT(signer)) = LOWER(UNACCENT(staff.full_name))` where staff.id = p_staff_id, OR
  - user has role 'Quản trị hệ thống'
  - Otherwise: can_sign=FALSE reason='Bạn không có quyền ký văn bản này (không phải người tạo/người ký)'
- For 'handling': JOIN handling_docs.signer (INT). Allow if `p_staff_id = signer`. Else: can_sign=FALSE reason='Chỉ người được giao ký HSCV mới ký được'.
- Return file_path + file_name for caller (avoid second query in route).

Use `unaccent` for Vietnamese case-insensitive name comparison (extension already enabled per init/01_create_schemas.sql). If unaccent not available, fallback to LOWER only.

**Part 4: fn_sign_transaction_list_by_staff** — Phase 11+12 list endpoint.
Signature: `(p_staff_id INT, p_tab VARCHAR, p_page INT, p_page_size INT)` returns `TABLE(id BIGINT, provider_code VARCHAR, provider_name VARCHAR, attachment_id BIGINT, attachment_type VARCHAR, file_name VARCHAR, doc_id BIGINT, doc_type VARCHAR, doc_label TEXT, status VARCHAR, error_message TEXT, created_at TIMESTAMPTZ, completed_at TIMESTAMPTZ, total_count BIGINT)`.

`p_tab` values: `'pending'` (status='pending'), `'completed'` (status='completed'), `'failed'` (status IN ('failed','expired','cancelled')). Invalid tab → empty result.

Body:
- FROM edoc.sign_transactions st WHERE st.staff_id = p_staff_id AND matching status set
- LEFT JOIN 4 attachment tables via CASE st.attachment_type to get file_name
- LEFT JOIN parent docs via st.doc_id + st.doc_type to get doc_label (VD: 'VB đi số 123/2026' or 'HSCV: Hợp đồng ABC')
- LEFT JOIN public.signing_provider_config spc ON spc.provider_code = st.provider_code to get provider_name
- Include total_count via COUNT(*) OVER()
- ORDER BY st.created_at DESC
- LIMIT p_page_size OFFSET ((p_page-1)*p_page_size)

NOTE on "Cần ký" tab: NOT in this SP — that tab queries VB/HSCV directly (not sign_transactions). This SP handles 3 tabs that come from `sign_transactions`.

**Part 5: fn_sign_transaction_count_by_staff** — badge count for tabs.
Signature: `(p_staff_id INT)` returns `TABLE(pending_count BIGINT, completed_count BIGINT, failed_count BIGINT)`.

Body: single SELECT with 3 COUNT(*) FILTER clauses against sign_transactions WHERE staff_id = p_staff_id. failed_count = sum of failed+expired+cancelled statuses.

```typescript
/**
 * Pure helpers for sign flow — no I/O, no DB. Easy to unit test.
 */

import type { SignAttachmentType } from '../../repositories/sign-transaction.repository.js';

const VALID_TYPES: SignAttachmentType[] = ['incoming', 'outgoing', 'drafting', 'handling'];

export function isValidAttachmentType(value: unknown): value is SignAttachmentType {
  return typeof value === 'string' && VALID_TYPES.includes(value as SignAttachmentType);
}

/**
 * Build MinIO key cho file đã ký.
 * Pattern: `signed/{type}/{txnId}/{original_basename}`
 *
 * VD: originalKey='documents/outgoing/2026/quyet-dinh-42.pdf', txnId=99
 *  → 'signed/outgoing/99/quyet-dinh-42.pdf'
 *
 * Giữ nguyên original key bên cạnh (không overwrite) — audit trail + rollback
 * nếu ký lại cần file gốc.
 */
export function buildSignedObjectKey(
  originalKey: string,
  attachmentType: SignAttachmentType,
  signTransactionId: number,
): string {
  if (!originalKey || typeof originalKey !== 'string') {
    throw new Error('originalKey không hợp lệ');
  }
  if (!Number.isFinite(signTransactionId) || signTransactionId <= 0) {
    throw new Error('signTransactionId phải là số nguyên dương');
  }

  // Lấy basename (phần sau dấu / cuối cùng)
  const parts = originalKey.split('/');
  const basename = parts[parts.length - 1] || 'signed.pdf';

  return `signed/${attachmentType}/${signTransactionId}/${basename}`;
}

/**
 * Extract human-readable file extension cho modal display.
 * VD: 'report.pdf' → 'pdf'
 */
export function getFileExtension(fileName: string): string {
  const dot = fileName.lastIndexOf('.');
  return dot === -1 ? '' : fileName.slice(dot + 1).toLowerCase();
}
```

## Trust Boundaries

|----------|-------------|
| route → SP (can_sign) | staff_id from JWT reaches SP for ACL check — tampering here bypasses permission |
| worker → SP (finalize_sign) | worker runs with full DB creds; attachment_id comes from queue job payload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | T (Tampering) | fn_attachment_can_sign | mitigate | Route layer passes `staffId` from `req.user.staffId` JWT only — never from body. Verified in Plan 03. |
| T-11-02 | E (Elevation) | fn_attachment_can_sign | mitigate | SP uses `UNACCENT+LOWER` comparison of doc.signer vs staff.full_name; falls through to `created_by` or admin role. Cannot be bypassed by attachment ID guessing. |

| T-11-04 | S (Spoofing) | fn_attachment_finalize_sign | accept | Worker process trusted (runs in-backend, no external caller). DoS via fake txn_id mitigated by Plan 04 double-complete guard in fn_sign_transaction_complete. |
