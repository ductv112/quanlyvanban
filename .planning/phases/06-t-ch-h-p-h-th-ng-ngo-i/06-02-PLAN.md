2. Create `routes/lgsp.ts`:

LGSP Token management via Redis (per D-01):
- Helper function `getValidToken(unitId)`: check Redis key `lgsp:token:{unitId}`, if expired call `lgspService.login()` or `refreshToken()`, cache new token with TTL 3300s (55 min, before 1hr expiry).

Endpoints:
- `GET /` — list LGSP documents: `lgspRepository.getDocumentList(req.user.unitId, direction, status, search, page, pageSize)`
- `GET /:id` — get document detail: `lgspRepository.getDocumentById(id)`
- `POST /send` — send outgoing doc via LGSP: get valid token -> `lgspService.sendEdoc()` -> `lgspRepository.createDocument()` with direction='send'
- `POST /receive` — manually trigger receive (also used by worker): get valid token -> `lgspService.getReceivedEdocs()` -> for each: `lgspService.getEdocContent()` -> parse edXML -> `lgspRepository.createDocument()` with direction='receive'
- `PATCH /:id/status` — update LGSP doc status
- `GET /agencies` — list synced agencies: `lgspRepository.getAgencyList()`
- `POST /agencies/sync` — sync agencies from LGSP: get valid token -> `lgspService.getAgencies()` -> for each: `lgspRepository.upsertAgency()`
