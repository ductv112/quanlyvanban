## Page Inventory

### 1. Trang Văn bản liên thông — `/van-ban-lien-thong`

**Pattern:** Reuse VB đến list pattern exactly.

Layout: `.page-header` + `.page-card` wrapping `<Table>`.

Columns (in order):
1. STT — 60px, center
2. Ngày nhận — 110px
3. Ký hiệu — 160px, bold `#1B3A5C`
4. Trích yếu — flexible, ellipsis with Tooltip
5. Hạn trả lời — 110px, red text if overdue (`#DC2626`)
6. Đơn vị phát hành — 180px
7. Người ký — 140px
8. Trạng thái — 120px, Tag component

Filter row (`.filter-row`): Input tìm kiếm (keyword) + Select Loại VB + Select Trạng thái + RangePicker ngày nhận.

Row click: navigate to detail page `/van-ban-lien-thong/[id]`.

### 2. Trang chi tiết VB liên thông — `/van-ban-lien-thong/[id]`

**Pattern:** Reuse VB đến detail pattern.

Layout: `.detail-header` (toolbar) + card tabs.

Toolbar actions (`.detail-header-right`):
- "Nhận bàn giao" — `<Button type="primary">` success color — Popconfirm: "Bạn có chắc chắn nhận bàn giao văn bản này?"
- "Chuyển lại" — `<Button>` default — `Modal.confirm` với TextArea lý do (required)
- "Hủy duyệt" — `<Button danger>` — Popconfirm: "Bạn có chắc chắn hủy duyệt văn bản này? Thao tác này không thể hoàn tác."

Info display: `.info-grid` (2 columns), `.doc-abstract-box` for trích yếu, attachment list `.attachment-item`.

### 3. Nút "Giao việc" trên VB đến detail toolbar

**Pattern:** Add button to existing VB đến detail page toolbar.

- Icon: `<ThunderboltOutlined />` (biểu trưng hành động nhanh)
- Label: "Giao việc"
- Type: `<Button type="primary" icon={<ThunderboltOutlined />}>` — accent teal `#0891B2`
- Opens: Drawer width 720px, `rootClassName="drawer-gradient"`

Giao việc Drawer fields (pre-filled from VB đến):
1. Tên hồ sơ (required) — pre-fill từ trích yếu VB
2. Hạn xử lý (required) — pre-fill từ hạn trả lời VB
3. Người phụ trách (required) — Select nhân viên (multi-select)
4. Ghi chú — TextArea, optional

Footer: "Tạo và giao việc" (primary) | "Hủy" (default)

### 4. Trang Tin nhắn — `/tin-nhan`

**Pattern:** Mail-like 3-panel layout (Gmail-inspired).

Layout structure (CSS class: `.mail-layout`):
```
[sidebar 220px] | [message-list 360px] | [message-detail flex-1]
```

Mail sidebar (dark panel, background `#0F1A2E`):
- "Soạn tin nhắn" button — full-width, `type="primary"`, icon `<EditOutlined />`
- Separator
- "Hộp thư đến" + Badge (unread count, `#0891B2`) — active state: background `#1B3A5C`
- "Đã gửi"
- "Thùng rác"

Message list panel (background `#FFFFFF`, border-right: `1px solid #E8ECF1`):
- Search input at top (keyword filter)
- Each message row: `.mail-item`
  - Avatar (32px) left
  - Sender name (14px 600) + timestamp (12px `#64748B`) right
  - Subject (14px) + preview snippet (12px `#64748B`, 1 line ellipsis)
  - Unread: left border `4px solid #0891B2`, background `#F0F7FF`, sender name bold
  - Read: no left border, background `#FFFFFF`
  - Hover: background `#F8FAFC`
  - Selected: background `#EFF8FF`, border-left `4px solid #1B3A5C`

Message detail pane (background `#FFFFFF`, padding 24px):
- Subject: 22px 700 `#1B3A5C`, margin-bottom 8px
- From/To meta: 12px `#64748B`
- Body: 14px 400 line-height 1.6, `#334155`
- Thread replies: styled as `.opinion-item` (reuse existing pattern)
- Reply textarea at bottom: TextArea + "Trả lời" button (`type="primary"`)

Empty state (no message selected): `.empty-center` with `<MailOutlined style={{ fontSize: 48, color: '#CBD5E1' }} />` + "Chọn tin nhắn để xem"

### 5. Drawer Soạn tin nhắn

Width: 720px, `rootClassName="drawer-gradient"`
Title: "Soạn tin nhắn mới"

Fields:
1. Người nhận (required) — `<Select mode="multiple">` tìm kiếm nhân viên
2. Tiêu đề (required) — `<Input>` maxLength 200
3. Nội dung (required) — `<Input.TextArea>` rows={6}, plain text (no rich text — auto mode default)

Footer: "Gửi tin nhắn" (primary, icon `<SendOutlined />`) | "Hủy" (default)

### 6. Trang Thông báo — `/thong-bao`

Layout: `.page-header` + `.page-card` wrapping List or Table.

Display as List (`<List>` component), not Table.

Each notification item:
- Icon left: `<BellOutlined />` in 32px circle (background `#EFF8FF`, color `#0891B2`)
- Title (14px 600) + timestamp (12px `#64748B`)
- Body snippet (14px `#64748B`, 2 lines ellipsis)
- Unread: bold title, background `#F0F7FF`, dot indicator `#0891B2`
- Read: normal weight, background `#FFFFFF`
- Hover: background `#F8FAFC`

Filter: Tabs — "Tất cả" | "Chưa đọc" | "Đã đọc"

Action: "Đánh dấu đã đọc tất cả" — `<Button>` default, top-right of card.

Admin: "Tạo thông báo" — `<Button type="primary">` top-right, opens Drawer (Người nhận multi-select + Tiêu đề + Nội dung).

### 7. Bell icon Dropdown (MainLayout header)

Existing: `<Badge count={3} size="small"><BellOutlined /></Badge>` — upgrade to functional dropdown.

Dropdown overlay: `<Dropdown>` component, placement `bottomRight`, trigger `['click']`.

Overlay card (background `#FFFFFF`, border-radius 12px, box-shadow level 2):
- Header: "Thông báo" (16px 600 `#1B3A5C`) + "Đánh dấu tất cả đã đọc" link (12px `#0891B2`)
- Max 10 notification items, each: icon + title (1 line) + timestamp (12px)
- Unread items: background `#F0F7FF`, dot left
- "Xem tất cả thông báo" link at bottom → `/thong-bao`
- Max-height: 480px, overflow-y: auto
- Width: 360px

## State & Interaction Map

### VB liên thông list
| State | Interaction | Visual |
|-------|-------------|--------|
| Loading | fetchData in-flight | `<Table loading={true}>` |
| Empty | No records match filter | `<Table locale={{ emptyText: <Empty description="Không có văn bản liên thông" /> }}>` |
| Row hover | Mouse over | background `#F0F7FF` (theme: rowHoverBg) |
| Row click | Navigate to detail | `router.push('/van-ban-lien-thong/[id]')` |

### Giao việc drawer
| State | Visual |
|-------|--------|
| Initial open | Fields pre-filled from VB data, cursor on "Người phụ trách" |
| Saving | Save button shows `loading={true}`, disabled |
| Success | Drawer closes, `message.success('Giao việc thành công')` |
| Error (server) | `message.error(errorMsg)` |
| Validation fail | Inline form error below each field |

### Tin nhắn
| State | Visual |
|-------|--------|
| Loading list | `<Skeleton active avatar paragraph={{ rows: 2 }} />` × 5 |
| No messages | `.empty-center` + MailOutlined icon + "Chưa có tin nhắn nào" |
| Sending | "Gửi tin nhắn" button `loading={true}` |
| Send success | Drawer closes, `message.success('Gửi tin nhắn thành công')`, inbox refreshes |
| New message via Socket | `message.info('Bạn có tin nhắn mới từ {tên}')` + badge count +1 |

### Thông báo bell
| State | Visual |
|-------|--------|
| No unread | `<Badge count={0}>` — dot hidden |
| Has unread | `<Badge count={N} size="small">` — orange dot `#D97706` |
| New via Socket | Badge count animates +1, `message.info` toast |
| Dropdown open | Overlay 360px × max 480px slides down |
| Mark all read | Badge count resets to 0, all items lose unread style |

## Socket.IO Visual Contract

When Socket.IO connection established:
- No visual indicator needed (silent background connection)

When connection lost:
- No error alert unless a user action fails — let action-level errors surface naturally

Event → UI mapping:
| Event | Visual Response |
|-------|----------------|
| `new_document` | Bell badge +1, `message.info('Có văn bản mới cần xử lý')` |
| `new_message` | Hộp thư đến badge +1, `message.info('Bạn có tin nhắn mới')` |
| `new_notification` | Bell badge +1, `message.info(notification.title)` |
| `doc_status_changed` | No toast (background update) — relevant page refreshes on next load |

All `message.*` toasts use `App.useApp()` pattern (not top-level `message` import).
Toast duration: default 3s.
