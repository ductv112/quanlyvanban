## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label spacing |
| sm | 8px | Between form fields (compact), tab item gaps |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding (`.main-content`, `.drawer-gradient .ant-drawer-body`) |
| xl | 32px | Between major sections on detail page |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Page-level top margin (loading skeleton) |

Exceptions:
- Touch target minimum 44px height for all action buttons on detail toolbar (HSCV-07 state buttons)
- Filter tab bar: Ant Design `Tabs` default 40px height — acceptable, do not override
- Progress bar column in table: 120px fixed width, 8px height (compact inline)
- Comment/opinion avatar: 32px circle

Source: `theme.ts` `controlHeight: 36`, `padding: 16`, `paddingLG: 24`

### 3. Trang Chi tiết HSCV — `/ho-so-cong-viec/:id` (HSCV-03)

**Detail Header Bar (`.detail-header`):**

**Card Tabs:**
- Component: Ant Design `Tabs` với `type="card"` bên trong `Card`

  1. `FolderOutlined` **Thông tin chung** — info-grid layout
  2. `FileTextOutlined` **Văn bản liên kết** — bảng VB + nút Thêm VB
  3. `TeamOutlined` **Cán bộ xử lý** — Transfer panel phân công
  4. `CommentOutlined` **Ý kiến xử lý** — comment feed + form thêm ý kiến
  5. `PaperClipOutlined` **File đính kèm** — upload + danh sách file
  6. `ApartmentOutlined` **HSCV con** — bảng HSCV con + nút Tạo HSCV con

**Tab 1 — Thông tin chung:**
- Dùng `.info-grid` (2 cột) cho: Ngày mở, Hạn giải quyết, Lĩnh vực, Loại VB, Quy trình, Trạng thái
- Dùng `.info-grid-full` cho: Ghi chú (nếu có — trong `.doc-abstract-box` style)
- Section title `.section-title` trên mỗi nhóm thông tin

**Tab 2 — Văn bản liên kết:**
- Header: tiêu đề "Văn bản liên kết" + `Badge count` + nút "Thêm văn bản" (primary ghost)
- Table: Số VB | Trích yếu | Loại | Ngày ký | Thao tác (nút Gỡ liên kết)
- Modal tìm kiếm VB: `Input.Search` + Tabs loại (VB đến / VB đi / Dự thảo) + Table kết quả với checkbox chọn nhiều

**Tab 3 — Cán bộ xử lý:**
- Transfer panel custom: Left = tree đơn vị (`DirectoryTree`, dùng `tree-utils.ts`) + danh sách cán bộ theo đơn vị chọn; Right = danh sách cán bộ đã phân công
- Mỗi mục trong danh sách phải chọn: `Radio.Group` "Phụ trách" / "Phối hợp" + `DatePicker` Hạn xử lý
- Phụ trách: badge `#059669`, Phối hợp: badge `#0891B2`
- Nút "Lưu phân công" (primary) ở cuối panel

**Tab 4 — Ý kiến xử lý:**
- Comment feed: dùng Ant Design `List` với custom render. Mỗi item: avatar tròn 32px (chữ cái đầu tên, màu ngẫu nhiên stable từ id), tên cán bộ (font-weight 600), timestamp (12px muted), nội dung ý kiến

- Phân tách bằng `Divider` giữa feed và form

**Tab 5 — File đính kèm:**
- `Upload.Dragger` cho drag-and-drop upload, maxSize 50MB, accept `.pdf,.doc,.docx,.xls,.xlsx,.png,.jpg`
- Danh sách file: icon loại file + tên file + kích thước + ngày upload + nút Tải xuống + nút Xóa

**Tab 6 — HSCV con:**
- Nút "Tạo HSCV con" (primary ghost) ở header
- Table HSCV con: cột giống danh sách chính nhưng compact, không có cột phụ trách/lãnh đạo ký

## Interaction Contracts

### Navigation
- Click tên HSCV trong table → navigate to `/ho-so-cong-viec/:id` (push, NOT new tab)
- `ArrowLeftOutlined` trên detail page → `router.back()` (không hardcode path)
- Breadcrumb: Trang chủ / Hồ sơ công việc / {Tên HSCV}

### Loading States
- Page load: `<Skeleton active paragraph={{ rows: 6 }} />` bên trong `.page-card`
- Detail page load: Skeleton cho `.detail-header` + 2 card sections
- Table: `loading={isLoading}` prop trực tiếp trên `<Table>`
- KPI cards: `<Skeleton.Avatar active size={48} />` + `<Skeleton.Input active size="small" />`
- Workflow canvas: `<Spin size="large" />` centered trong canvas container (duy nhất nơi dùng `<Spin>`)
- Không dùng `<Spin />` full-page

### Form Validation
- `validateTrigger="onSubmit"` trên tất cả Drawer forms để tránh layout shift
- Hiển thị lỗi inline bên dưới field (Ant Design Form.Item default)
- Backend validation errors: map về field cụ thể nếu có, fallback `message.error(...)` nếu không map được

### Delete Pattern
- Tất cả xóa dùng `Modal.confirm` với `type="warning"`, `okType="danger"`, `okText="Xóa"`, `cancelText="Hủy bỏ"`
- Không dùng `Popconfirm` cho xóa (chỉ dùng Popconfirm cho các confirm nhẹ như lock/unlock)

### Workflow Transition Pattern
- Các action "Từ chối" và "Trả về" yêu cầu lý do bắt buộc — không cho submit nếu trống
- Action "Duyệt" và "Hoàn thành" không cần xác nhận thêm nếu không phá hủy data
- Sau transition thành công: reload lại trang chi tiết (re-fetch status + toolbar)

### File Upload
- Feedback ngay khi bắt đầu upload: progress bar trong Upload component
- Lỗi upload: hiển thị tên file + thông báo lỗi cụ thể (quá dung lượng, sai định dạng)
- Xóa file: Popconfirm nhẹ "Xóa file này?"

### ReactFlow Designer
- Node drag: snap-to-grid 8px
- Double-click node → mở Node Properties panel
- Delete node: phím `Delete`/`Backspace` khi node đang selected + confirmation nếu có connections
- Lưu quy trình: Button "Lưu quy trình" (primary) → success toast
