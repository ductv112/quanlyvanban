<objective>
Migrate data từ legacy column `public.staff.sign_phone` (đã tồn tại từ v1.0) sang bảng mới `public.staff_signing_config` với `provider_code='SMARTCA_VNPT'`, sau đó DROP column cũ.

**Why:** v1.0 đã ship với mock signing dùng `staff.sign_phone` lưu số điện thoại SmartCA. v2.0 chuyển sang schema multi-provider; phải migrate data tồn đọng để không user nào phải cấu hình lại.

**Strategy (ATOMIC migration trong 1 transaction):**
1. Start transaction
2. INSERT INTO staff_signing_config SELECT ... FROM staff WHERE sign_phone IS NOT NULL AND sign_phone != ''
3. Verify count match (raise exception nếu mismatch)
4. DROP COLUMN staff.sign_phone
5. COMMIT

**Fallback:** Nếu production có thể cần rollback, column cũ được giữ 1 ngày qua soft-deprecation comment — NHƯNG theo spec v2.0 (MIG-04) drop thẳng sau verify. Nếu DB chưa có user nào set sign_phone, migration chỉ DROP column (no-op cho INSERT).

Output: 1 file SQL idempotent — chạy 2 lần không fail (2nd run: column đã drop, INSERT no-op).
</objective>

```sql
-- ============================================================================
-- Migration 041: Migrate staff.sign_phone → staff_signing_config + DROP column
-- Requirements: MIG-03, MIG-04
-- Depends on: 040_signing_schema.sql (staff_signing_config table exists)
-- Idempotent: chạy 2 lần không fail (2nd run: column đã drop → skip INSERT + DROP no-op)
-- ============================================================================

DO $$
DECLARE
  v_column_exists BOOLEAN;
  v_source_count  INT := 0;
  v_target_count  INT := 0;
  v_inserted      INT := 0;
BEGIN
  -- Kiểm tra cột sign_phone còn tồn tại không
  SELECT EXISTS (
    SELECT 1 FROM information_schema.columns
    WHERE table_schema = 'public'
      AND table_name   = 'staff'
      AND column_name  = 'sign_phone'
  ) INTO v_column_exists;

  IF NOT v_column_exists THEN
    RAISE NOTICE '⚠️  Migration 041: Cột staff.sign_phone đã bị drop — skip (idempotent)';
    RETURN;
  END IF;

  -- ======== Count source rows có sign_phone non-empty ========
  EXECUTE 'SELECT COUNT(*) FROM public.staff
           WHERE sign_phone IS NOT NULL AND TRIM(sign_phone) != '''''
    INTO v_source_count;

  RAISE NOTICE '📊 Source: % staff có sign_phone non-empty', v_source_count;

  -- ======== Copy data sang staff_signing_config ========
  -- Dùng EXECUTE vì PL/pgSQL parse sign_phone column tại compile time;
  -- nếu chạy 2nd time (column đã drop), PL/pgSQL sẽ fail parse — phải dùng dynamic SQL
  EXECUTE '
    INSERT INTO public.staff_signing_config
      (staff_id, provider_code, user_id, is_verified, created_at, updated_at)
    SELECT s.id,
           ''SMARTCA_VNPT''::VARCHAR,
           s.sign_phone::VARCHAR,
           FALSE,
           NOW(),
           NOW()
    FROM public.staff s
    WHERE s.sign_phone IS NOT NULL
      AND TRIM(s.sign_phone) != ''''
    ON CONFLICT (staff_id, provider_code) DO NOTHING
  ';

  GET DIAGNOSTICS v_inserted = ROW_COUNT;
  RAISE NOTICE '✅ Inserted % rows vào staff_signing_config', v_inserted;

  -- ======== Verify count match ========
  SELECT COUNT(*) FROM public.staff_signing_config
    WHERE provider_code = 'SMARTCA_VNPT'
    INTO v_target_count;

  RAISE NOTICE '📊 Target count (staff_signing_config WHERE provider=SMARTCA_VNPT): %', v_target_count;

  -- Target phải >= source (có thể > nếu user đã manual tạo config qua API mới trước migration)
  IF v_target_count < v_source_count THEN
    RAISE EXCEPTION 'Migration FAILED: target count (%) < source count (%) — rollback', v_target_count, v_source_count;
  END IF;

  -- ======== DROP column staff.sign_phone ========
  ALTER TABLE public.staff DROP COLUMN IF EXISTS sign_phone;
  RAISE NOTICE '✅ DROP COLUMN public.staff.sign_phone — hoàn tất';

  -- Note: sign_ca và sign_image KHÔNG drop (theo spec MIG-04 chỉ drop sign_phone)
  -- Lý do giữ: sign_ca (chứng thư số) có thể cần cho hiển thị subject; sign_image (ảnh chữ ký scan) dùng cho stamp

  RAISE NOTICE '✅ Migration 041 HOÀN TẤT: % rows migrated, cột sign_phone đã drop', v_inserted;
END $$;

DO $$ BEGIN
  RAISE NOTICE '✅ Migration 041: staff.sign_phone → staff_signing_config + DROP column';
END $$;
```

**Decisions lock-in:**
- Dùng `EXECUTE` dynamic SQL để idempotent (chạy 2nd time: cột đã drop → PL/pgSQL static SELECT sẽ fail parse; dynamic thì bỏ qua vì check v_column_exists ở đầu).
- `ON CONFLICT DO NOTHING` — nếu migration chạy 2 lần do race, không duplicate.
- `is_verified=FALSE` — user phải re-verify với provider thật trong Phase 10.
- `sign_ca` và `sign_image` **giữ nguyên** trong `staff` table (out of scope MIG-04).
- Nếu `v_target_count < v_source_count` → RAISE EXCEPTION → toàn bộ DO block rollback → cột sign_phone KHÔNG bị drop → safe.

**QUAN TRỌNG:** Chạy migration vào qlvb_postgres ngay sau khi tạo file. Nếu DB hiện tại không có user nào set sign_phone, script chạy OK và drop column (v1.0 seed có thể có hoặc không — verify bằng SELECT trước).
