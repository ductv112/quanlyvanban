**Bước 2 — Thêm handler `GET /:id/download` NGAY TRƯỚC `GET /:id`:**

Quan trọng: Express match route theo thứ tự đăng ký. Vì cả `/:id` và `/:id/download` có dạng tương tự nhưng độ dài path khác nhau, Express phân biệt được qua path segment — cả 2 OK. Tuy nhiên theo D-03 trong CONTEXT, đặt thứ tự: `POST /` → `POST /:id/cancel` → `GET /:id/download` → `GET /:id` để rõ ràng (handler cụ thể trước, catch-all sau).

Chèn khối code sau giữa kết thúc handler `POST /:id/cancel` (dòng ~387) và bắt đầu handler `GET /:id` (dòng ~392):

```typescript
// ============================================================================
// GET /:id/download — Presigned URL cho file PDF đã ký (owner-or-admin only)
// Response: { success: true, data: { url, file_name, expires_in: 600 } }
// TTL 600s (10 phút), header Cache-Control: no-store (T-12-06).
// ============================================================================
router.get('/:id/download', async (req: Request, res: Response) => {
  try {
    const { staffId, isAdmin } = (req as AuthRequest).user;
    const id = Number(req.params.id);
    if (!Number.isFinite(id) || id <= 0) {
      res.status(400).json({ success: false, message: 'ID không hợp lệ' });
      return;
    }

    const txn = await signTransactionRepository.getById(id);
    if (!txn) {
      res.status(404).json({
        success: false,
        message: 'Không tìm thấy giao dịch ký số',
      });
      return;
    }
    // SECURITY T-12-01: owner-or-admin only (mitigate info disclosure)
    if (txn.staff_id !== staffId && !isAdmin) {
      res.status(403).json({
        success: false,
        message: 'Bạn không có quyền tải file của giao dịch này',
      });
      return;
    }
    // SECURITY T-12-02: chỉ cho download khi đã hoàn tất + có signed file
    if (txn.status !== 'completed' || !txn.signed_file_path) {
      res.status(404).json({
        success: false,
        message: 'Giao dịch chưa có file đã ký',
      });
      return;
    }

    // Presigned URL TTL 600s
    const url = await getFileUrl(txn.signed_file_path, 600);

    // T-12-06: Ngăn browser cache URL có signature
    res.setHeader('Cache-Control', 'no-store');

    // file_name: prefix 'signed_' vào tên từ segment cuối của signed_file_path
    const segments = txn.signed_file_path.split('/');
    const lastSegment = segments[segments.length - 1] || 'signed.pdf';
    const fileName = lastSegment.startsWith('signed_') ? lastSegment : `signed_${lastSegment}`;

    res.json({
      success: true,
      data: {
        url,
        file_name: fileName,
        expires_in: 600,
      },
    });
  } catch (error) {
    handleDbError(error, res);
  }
});

```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → `/api/ky-so/sign/:id/download` | JWT-authenticated request — client có thể gửi bất kỳ `id` số nào |
| BE → MinIO | Internal network (trust boundary thấp); MinIO PRIVATE bucket policy giả định (verify Phase 14) |
| BE → client (response) | Trả presigned URL có chữ ký HMAC — URL này cho phép bất kỳ ai có link download trong 600s |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | I (Info Disclosure) | GET /:id/download handler | mitigate | Check `txn.staff_id === req.user.staffId` OR `req.user.isAdmin`; nếu không → 403 Vietnamese message, KHÔNG gọi MinIO. Test: curl với JWT user khác → 403. |
| T-12-02 | I (Info Disclosure) | GET /:id/download handler | mitigate | Check `txn.status === 'completed'` AND `txn.signed_file_path IS NOT NULL` trước khi tạo presigned URL; nếu không → 404 "Giao dịch chưa có file đã ký". Ngăn leak URL của txn pending/failed. |
| T-12-03 | D (DoS) | GET /:id/download endpoint | accept | Rate limit deferred Phase 14 — low risk cho v2.0 demo (authenticated endpoint, staff nội bộ, không public). Document trong comment header file. |
| T-12-06 | I (Info Disclosure) | HTTP response caching | mitigate | Set `res.setHeader('Cache-Control', 'no-store')` trên response — ngăn browser/proxy cache URL có HMAC signature; URL leak sau 600s không còn valid dù sao. |
| T-12-07 | T (Tampering) | `id` param | mitigate | `Number.isFinite(id) && id > 0` check — reject NaN / negative / zero. DB FK tiếp tục enforce tồn tại. Pattern khớp handler cancel + GET /:id hiện có. |

**Out of scope** (Phase 14):
- Rate limit (T-12-03)
- Audit log MongoDB non-repudiation (`{ staff_id, txn_id, action: 'download_signed', ts, ip, ua }`)
- MinIO public URL tách internal vs external (nếu BE qua `minio:9000` nhưng FE cần `https://s3.domain`)
