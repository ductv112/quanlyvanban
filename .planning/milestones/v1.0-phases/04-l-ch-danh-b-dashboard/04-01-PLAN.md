Create migration file `014_sprint9_calendar_directory.sql` with:

**Table: `public.calendar_events`**
- `id BIGSERIAL PRIMARY KEY`
- `title VARCHAR(500) NOT NULL`
- `description TEXT`
- `start_time TIMESTAMP NOT NULL`
- `end_time TIMESTAMP NOT NULL`
- `all_day BOOLEAN DEFAULT false`
- `color VARCHAR(20) DEFAULT '#1B3A5C'` — hex color for event card (per D-03)
- `repeat_type VARCHAR(20) DEFAULT 'none'` — values: none, daily, weekly, monthly
- `scope VARCHAR(20) NOT NULL DEFAULT 'personal'` — values: personal, unit, leader
- `unit_id INT` — for scope=unit/leader, references departments(id)
- `created_by INT NOT NULL REFERENCES staff(id)`
- `created_at TIMESTAMP DEFAULT NOW()`
- `updated_at TIMESTAMP DEFAULT NOW()`
- `is_deleted BOOLEAN DEFAULT false`
- INDEX on `(scope, unit_id, start_time)`
- INDEX on `(created_by, start_time)`

**Stored Functions:**

1. `public.fn_calendar_event_get_list(p_scope VARCHAR, p_unit_id INT, p_staff_id INT, p_start DATE, p_end DATE)`
   - Returns events filtered by scope + date range
   - For scope='personal': filter by created_by = p_staff_id
   - For scope='unit': filter by unit_id = p_unit_id
   - For scope='leader': filter by unit_id = p_unit_id AND scope='leader'
   - WHERE is_deleted = false AND start_time between p_start and p_end
   - RETURNS TABLE(id, title, description, start_time, end_time, all_day, color, repeat_type, scope, unit_id, created_by, creator_name)
   - JOIN staff for creator_name

2. `public.fn_calendar_event_get_by_id(p_id BIGINT)`
   - Returns single event with creator info
   - RETURNS TABLE(id, title, description, start_time, end_time, all_day, color, repeat_type, scope, unit_id, created_by, creator_name, created_at, updated_at)

3. `public.fn_calendar_event_create(p_title, p_description, p_start_time, p_end_time, p_all_day, p_color, p_repeat_type, p_scope, p_unit_id, p_created_by)`
   - INSERT into calendar_events
   - RETURNS TABLE(success BOOLEAN, message TEXT, id BIGINT)

4. `public.fn_calendar_event_update(p_id, p_title, p_description, p_start_time, p_end_time, p_all_day, p_color, p_repeat_type, p_scope, p_unit_id, p_staff_id)`
   - UPDATE calendar_events WHERE id = p_id
   - Check ownership: created_by = p_staff_id (for personal scope)
   - RETURNS TABLE(success BOOLEAN, message TEXT)

5. `public.fn_calendar_event_delete(p_id BIGINT, p_staff_id INT)`
   - Soft delete (is_deleted = true)
   - Check ownership for personal scope
   - RETURNS TABLE(success BOOLEAN, message TEXT)

6. `public.fn_directory_get_list(p_unit_id INT, p_department_id INT, p_search VARCHAR, p_page INT, p_page_size INT)`
   - Query staff table with JOIN departments, positions
   - Filter by unit_id, department_id (optional), search on full_name/phone/email (ILIKE)
   - WHERE is_locked = false
   - RETURNS TABLE(id, full_name, position_name, department_name, unit_name, phone, mobile, email, image, total_count)
   - Pagination via OFFSET/LIMIT

Follow the exact SQL style from 013 migration (BEGIN/COMMIT, DO $$ blocks, IF NOT EXISTS checks).
