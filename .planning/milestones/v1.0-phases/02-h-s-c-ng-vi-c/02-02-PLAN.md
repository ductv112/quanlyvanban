**Endpoints to create:**

```
GET /                    — getList (parse query params: filter_type, status, keyword, from_date, to_date, page, page_size)
GET /count-by-status     — countByStatus (for tab badges)
POST /                   — create (body: name, doc_type_id, doc_field_id, comments, start_date, end_date, curator_id, signer_id, workflow_id, is_from_doc, parent_id)
GET /:id                 — getById
PUT /:id                 — update
DELETE /:id              — delete

GET /:id/can-bo          — getStaff
POST /:id/phan-cong      — assignStaff (body: staff_ids[], role_type, deadline)
DELETE /:id/phan-cong/:staffId — removeStaff

GET /:id/y-kien          — getOpinions
POST /:id/y-kien         — createOpinion (body: content, opinion_type)

GET /:id/van-ban-lien-ket — getLinkedDocs
POST /:id/lien-ket-van-ban — linkDoc (body: doc_id, doc_type)
DELETE /:id/lien-ket-van-ban/:linkId — unlinkDoc

GET /:id/dinh-kem        — getAttachments
POST /:id/dinh-kem       — upload attachment (multer single 'file', save to MinIO bucket 'documents' with path `handling-docs/${docId}/${uuidv4()}-${originalname}`, insert via raw query or SP)
DELETE /:id/dinh-kem/:attachmentId — delete attachment (delete from MinIO + DB)

GET /:id/hscv-con        — getChildren

PATCH /:id/trang-thai    — changeStatus (body: action, reason)
  action values: 'submit' | 'approve' | 'reject' | 'return' | 'complete' | 'change'
  For 'reject' and 'return', reason is required — return 400 if empty.
  For 'change', body also includes new_status.

PATCH /:id/tien-do       — updateProgress (body: progress — validate 0-100)
```

Each handler: try/catch with handleDbError. Extract staffId/unitId/departmentId from (req as AuthRequest).user. Return `{ success: true, data: ... }` for reads, `{ success: true, message: '...', data: { id } }` for writes.

For attachment upload: use `upload.single('file')` middleware, then `uploadFile('documents', filePath, req.file.buffer, req.file.mimetype)`, then insert into attachment_handling_docs table via rawQuery (or create a simple SP). Return the created attachment record.

For attachment delete: get file_path from DB, call `deleteFile('documents', filePath)`, then delete DB row.
