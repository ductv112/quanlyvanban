**Route `workflow.ts`:**

```
GET /                   — getList (query: doc_field_id, is_active)
POST /                  — create (body: name, version, doc_field_id)
GET /:id                — getById (also return steps + links in one response)
PUT /:id                — update (body: name, version, doc_field_id, is_active)
DELETE /:id             — delete

POST /:id/steps         — createStep
PUT /steps/:stepId      — updateStep
DELETE /steps/:stepId   — deleteStep

POST /step-links        — createStepLink (body: from_step_id, to_step_id)
DELETE /step-links/:linkId — deleteStepLink

GET /steps/:stepId/staff   — getStepStaff
POST /steps/:stepId/staff  — assignStepStaff (body: staff_ids[])

GET /:id/full             — getById + getSteps + step links (combined response for designer)
```

For `GET /:id/full`: call getById, getSteps, then for each step pair query links. Return `{ flow, steps, links }`.

Step links: query via rawQuery `SELECT * FROM edoc.doc_flow_step_links WHERE from_step_id = ANY(step_ids) OR to_step_id = ANY(step_ids)` where step_ids are from the steps result.

All handlers: try/catch with handleDbError. Extract unitId from AuthRequest.
